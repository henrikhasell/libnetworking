libnetworking

Why I made libnetworking:

As a developer, I wanted to create a cross-platform, multi-player video game for the PC. I chose the C programming language because it is simple and easy to understand. I realised there was no networking library that fit my specific needs: Lightweight, minimalistic, cross-platform and UDP-based networking. Though networking libraries exist for C, I felt they were either too bloated, or did not contain features that I needed to have in my application: IPv6 compatibility, modularity and simplicity.

Why I did not use the Berkley Sockets API model:

It is implemented differently in Windows and UNIX. For further information, please see WinSock. I have made cross-platform applications with this API in the past, but it was a nightmare to work with. I reasoned it would be less work for me in the long run if I made my own compatibility layer for WinSock/UNIX to eliminate the differences between the API. This is in essence what libnetworking is.

This library is designed to address three issues:

  1. It removes the platform specific differences of the C BSD socket API, allowing for truly platform independent code.
  2. It provides a easy to use networking API fully compatible with IPv6 IP addresses.
  3. It contains a framework built from the ground up, designed for streaming information via UDP. This includes the ability to monitor and account for packet loss.

Networking Sub-system:

Networking_Initialise

int Networking_Initialise()

This function initialises all of the libraries components. It should be called before any other function from this library.

#include <stdio.h>
#include <stddef.h>
#include <networking.h>

int main(int argc char *argv[]){
     if(Networking_Initialise() < 0){
         printf("Failed to initialise the networking subsystem.\n");
         return EXIT_FAILURE;
     }
     Networking_Cleanup();
     return EXIT_SUCCESS;
}
